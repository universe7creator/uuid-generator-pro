Frequently Asked

What's the difference between UUID versions?
UUID v4 is purely random. UUID v7 includes a timestamp prefix, making them sortable by creation time — great for database primary keys. UUID v8 is vendor-specific and customizable.
Is the generation cryptographically secure?
Yes! We use crypto.getRandomValues() which provides cryptographically strong random numbers. This is suitable for security tokens and session IDs.
Can I use the API in my application?
Pro users get full API access with up to 1,000 IDs per request. Perfect for CI/CD pipelines, database seeding, and automated workflows.
